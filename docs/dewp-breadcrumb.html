Breadcrumb 핵심 사용 가이드

1. HTML 구조 - 필수 요소

Breadcrumb을 사용하기 위한 최소한의 HTML 구조입니다:

<!-- 기본 구조 -->
<nav class="dewp-breadcrumb" aria-label="Breadcrumb">
  <ol style="list-style:none; padding:0; margin:0; display:flex; flex-wrap:wrap;">
    <li class="dewp-breadcrumb-item">
      <a href="/">홈</a>
      <span class="dewp-breadcrumb-separator">/</span>
    </li>
    <li class="dewp-breadcrumb-item">
      <a href="/category">카테고리</a>
      <span class="dewp-breadcrumb-separator">/</span>
    </li>
    <li class="dewp-breadcrumb-item is-active" aria-current="page">
      현재 페이지
    </li>
  </ol>
</nav>

필수 요소:

  • dewp-breadcrumb: 최상위 컨테이너 클래스 (필수)
  • aria-label="Breadcrumb": 접근성을 위한 라벨 (필수)
  • <ol>: 순서 있는 목록 (필수, 스타일 초기화 필요: list-style:none; padding:0; margin:0; display:flex; flex-wrap:wrap;)
  • dewp-breadcrumb-item: 각 항목 클래스 (필수)
  • dewp-breadcrumb-separator: 구분자 클래스 (필수, 마지막 항목 제외)
  • is-active: 현재 페이지 표시 클래스 (필수, 마지막 항목에만)
  • aria-current="page": 현재 페이지 접근성 속성 (필수, 마지막 항목에만)

주의사항: Breadcrumb은 순수 CSS 컴포넌트입니다. JavaScript가 필요하지 않습니다.

3. 구분자 커스터마이징

구분자는 기본적으로 슬래시(/)를 사용하지만, 다른 문자나 아이콘으로 변경할 수 있습니다:

<!-- 기본 슬래시 -->
<span class="dewp-breadcrumb-separator">/</span>

<!-- 다른 문자 -->
<span class="dewp-breadcrumb-separator">></span>
<span class="dewp-breadcrumb-separator">·</span>
<span class="dewp-breadcrumb-separator">→</span>

<!-- 아이콘 사용 -->
<span class="dewp-breadcrumb-separator">
  <i class="icon-chevron-right"></i>
</span>

예시 3: 동적 생성 (JavaScript)
// 경로 배열로부터 브레드크럼 생성
function createBreadcrumb(items) {
  const nav = document.createElement('nav');
  nav.className = 'dewp-breadcrumb';
  nav.setAttribute('aria-label', 'Breadcrumb');
  
  const ol = document.createElement('ol');
  ol.style.cssText = 'list-style:none; padding:0; margin:0; display:flex; flex-wrap:wrap;';
  
  items.forEach((item, index) => {
    const li = document.createElement('li');
    li.className = 'dewp-breadcrumb-item';
    
    if (index === items.length - 1) {
      // 마지막 항목 (현재 페이지)
      li.classList.add('is-active');
      li.setAttribute('aria-current', 'page');
      li.textContent = item.text;
    } else {
      // 링크 항목
      const link = document.createElement('a');
      link.href = item.url;
      link.textContent = item.text;
      li.appendChild(link);
      
      const separator = document.createElement('span');
      separator.className = 'dewp-breadcrumb-separator';
      separator.textContent = '/';
      li.appendChild(separator);
    }
    
    ol.appendChild(li);
  });
  
  nav.appendChild(ol);
  return nav;
}

// 사용 예시
const breadcrumb = createBreadcrumb([
  { text: '홈', url: '/' },
  { text: '제품', url: '/products' },
  { text: '전자제품', url: '/products/electronics' },
  { text: '스마트폰', url: '/products/electronics/smartphone' }
]);

document.body.appendChild(breadcrumb);

5. 스타일링 가이드

Breadcrumb은 기본 스타일이 제공되지만, 필요에 따라 커스터마이징할 수 있습니다:

/* 항목 간격 조정 */
.dewp-breadcrumb ol {
  gap: var(--db-gap-xs); /* 또는 원하는 간격 */
}

/* 링크 색상 변경 */
.dewp-breadcrumb-item a {
  color: var(--db-text-primary);
  text-decoration: none;
}

.dewp-breadcrumb-item a:hover {
  color: var(--db-primary);
  text-decoration: underline;
}

/* 현재 페이지 스타일 */
.dewp-breadcrumb-item.is-active {
  color: var(--db-text-secondary);
}

/* 구분자 스타일 */
.dewp-breadcrumb-separator {
  margin: 0 var(--db-gap-xs);
  color: var(--db-text-tertiary);
}

6. 접근성 (A11y)

Breadcrumb은 웹 접근성을 고려하여 설계되었습니다:

  • 시맨틱 HTML: <nav>와 <ol> 요소를 사용하여 구조적 의미 제공
  • aria-label: aria-label="Breadcrumb"로 스크린 리더에 내비게이션 유형 명시
  • aria-current: 현재 페이지 항목에 aria-current="page" 속성 추가
  • 키보드 네비게이션: 링크 항목은 Tab 키로 포커스 가능
  • 스크린 리더: "Breadcrumb"로 시작하여 경로를 단계별로 읽어줌

권장 사항:

  • 각 링크에 명확한 텍스트 제공
  • 현재 페이지는 링크가 아닌 일반 텍스트로 표시
  • 구분자는 시각적으로만 표시하고 스크린 리더에는 읽히지 않도록 처리 (CSS로 숨김 가능)

7. 주의사항 및 팁

  • 순서: 항목은 항상 홈에서 시작하여 현재 페이지까지 순차적으로 나열해야 합니다.
  • 마지막 항목: 마지막 항목은 항상 현재 페이지를 나타내며, 링크가 아닌 일반 텍스트로 표시합니다.
  • 구분자: 마지막 항목 뒤에는 구분자를 추가하지 않습니다.
  • 반응형: flex-wrap:wrap을 사용하여 작은 화면에서 자동으로 줄바꿈됩니다.
  • URL 구조: Breadcrumb의 URL은 실제 페이지 구조와 일치해야 합니다.
  • 깊이 제한: 너무 많은 단계는 사용자 경험을 저하시킬 수 있으므로, 적절한 깊이를 유지하세요.
  • 동적 생성: SPA(Single Page Application)에서는 JavaScript로 동적으로 생성하는 것이 일반적입니다.
