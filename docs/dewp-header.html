Header 핵심 사용 가이드

1. HTML 구조 - 기본 사용법

Header를 사용하기 위한 기본 HTML 구조입니다:

<div class="dewp-header">
  <div class="dewp-header-content">
    <h1 class="dewp-header-title">페이지 제목</h1>
    <p class="dewp-header-description">페이지 설명</p>
  </div>
  <div class="dewp-header-actions">
    <button class="dewp-btn dewp-btn-outline-secondary">취소</button>
    <button class="dewp-btn dewp-btn-primary">저장</button>
  </div>
</div>

필수 요소:

  • dewp-header: 최상위 컨테이너 클래스 (필수)
  • dewp-header-content: 콘텐츠 영역 (필수, 제목과 설명 포함)
  • dewp-header-title: 제목 (권장, <h1>~<h6> 태그 사용)
  • dewp-header-description: 설명 (선택사항)
  • dewp-header-actions: 액션 버튼 영역 (선택사항)

주의사항: Header는 순수 CSS 컴포넌트입니다. JavaScript가 필요하지 않습니다.

2. 기본 헤더

<div class="dewp-header">
  <div class="dewp-header-content">
    <h1 class="dewp-header-title">플러그인 설정</h1>
    <p class="dewp-header-description">워드프레스 관리자용 헤더 컴포넌트 예시</p>
  </div>
  <div class="dewp-header-actions">
    <button class="dewp-btn dewp-btn-outline-secondary">미리보기</button>
    <button class="dewp-btn dewp-btn-primary">저장</button>
  </div>
</div>

3. 뒤로가기 버튼이 있는 헤더

상세 페이지나 하위 페이지에서 상위 페이지로 돌아갈 수 있는 뒤로가기 버튼을 추가할 수 있습니다:

<div class="dewp-header dewp-header-with-back">
  <button class="dewp-header-back-button" type="button">
    <i class="icon-chevron-left" aria-hidden="true"></i>
    목록으로
  </button>
  <div class="dewp-header-content">
    <h1 class="dewp-header-title">게시글 상세</h1>
    <p class="dewp-header-description">게시글 정보를 확인하고 세부 설정을 관리할 수 있는 페이지입니다.</p>
  </div>
  <div class="dewp-header-actions">
    <button class="dewp-btn dewp-btn-outline-secondary">수정</button>
    <button class="dewp-btn dewp-btn-primary">저장</button>
  </div>
</div>

필수 클래스:

  • dewp-header-with-back: 뒤로가기 버튼이 있는 헤더 (컨테이너에 추가)
  • dewp-header-back-button: 뒤로가기 버튼 클래스

4. 실제 사용 예시

예시 1: 설정 페이지 헤더
<div class="dewp-header">
  <div class="dewp-header-content">
    <h1 class="dewp-header-title">사이트 설정</h1>
    <p class="dewp-header-description">사이트의 기본 정보를 설정합니다.</p>
  </div>
  <div class="dewp-header-actions">
    <button class="dewp-btn dewp-btn-secondary" type="button">초기화</button>
    <button class="dewp-btn dewp-btn-primary" type="submit">저장</button>
  </div>
</div>
예시 2: 상세 페이지 헤더 (뒤로가기 포함)
<div class="dewp-header dewp-header-with-back">
  <button class="dewp-header-back-button" type="button" onclick="history.back()">
    <i class="icon-chevron-left" aria-hidden="true"></i>
    목록으로
  </button>
  <div class="dewp-header-content">
    <h1 class="dewp-header-title">사용자 상세</h1>
    <p class="dewp-header-description">사용자 ID: #12345</p>
  </div>
  <div class="dewp-header-actions">
    <button class="dewp-btn dewp-btn-outline-danger" type="button">삭제</button>
    <button class="dewp-btn dewp-btn-primary" type="button">수정</button>
  </div>
</div>
예시 3: 액션 버튼 없는 헤더
<div class="dewp-header">
  <div class="dewp-header-content">
    <h1 class="dewp-header-title">대시보드</h1>
    <p class="dewp-header-description">전체 통계를 한눈에 확인하세요.</p>
  </div>
</div>
예시 4: 제목만 있는 헤더
<div class="dewp-header">
  <div class="dewp-header-content">
    <h1 class="dewp-header-title">알림</h1>
  </div>
</div>

5. 버튼 스타일 가이드

헤더의 액션 버튼은 일반적으로 다음과 같은 스타일을 사용합니다:

<div class="dewp-header-actions">
  <!-- 보조 액션 (왼쪽) -->
  <button class="dewp-btn dewp-btn-outline-secondary">취소</button>
  <button class="dewp-btn dewp-btn-secondary">초기화</button>
  
  <!-- 주요 액션 (오른쪽) -->
  <button class="dewp-btn dewp-btn-primary">저장</button>
</div>

권장 사항:

  • 주요 액션은 Primary 버튼을 사용하세요.
  • 보조 액션은 Secondary 또는 Outline 버튼을 사용하세요.
  • 위험한 액션(삭제 등)은 Danger 버튼을 사용하세요.
  • 버튼 순서는 보조 액션을 왼쪽, 주요 액션을 오른쪽에 배치하세요.

6. 접근성 (A11y)

Header는 웹 접근성을 고려하여 설계되었습니다:

  • 시맨틱 HTML: 제목은 적절한 헤딩 레벨(<h1>~<h6>)을 사용하세요.
  • 아이콘: 아이콘에는 aria-hidden="true"를 추가하고, 버튼에 명확한 텍스트를 제공하세요.
  • 키보드 접근: 모든 버튼은 키보드로 접근 가능해야 합니다.
  • 포커스 관리: 뒤로가기 버튼은 명확한 포커스 스타일을 가져야 합니다.

권장 사항:

  • 페이지당 하나의 <h1>을 사용하고, 헤더 제목에 사용하세요.
  • 뒤로가기 버튼에는 명확한 텍스트를 제공하세요 (예: "목록으로", "뒤로").
  • 설명 텍스트는 간결하고 명확하게 작성하세요.

7. 주의사항 및 팁

  • 제목 필수: dewp-header-title은 필수 요소입니다.
  • 설명 선택: dewp-header-description은 선택사항이지만, 페이지의 목적을 명확히 하기 위해 권장됩니다.
  • 액션 버튼: 액션 버튼이 없어도 헤더는 정상적으로 작동합니다.
  • 뒤로가기 기능: 뒤로가기 버튼에 onclick="history.back()" 또는 라우터의 뒤로가기 함수를 연결하세요.
  • 반응형: 작은 화면에서는 액션 버튼이 세로로 쌓이거나 드롭다운으로 표시될 수 있습니다.
  • 일관성: 같은 애플리케이션 내에서는 헤더 스타일을 일관되게 유지하세요.
  • 버튼 순서: 보조 액션을 왼쪽, 주요 액션을 오른쪽에 배치하는 것이 일반적입니다.
